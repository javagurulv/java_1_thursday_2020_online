Изучаем Single Responsibility Principle.

Как вы видите, бизнес может добавлять и добавлять новые правила
для автоматического обнаружения мошенничества.

Подумайте о том, за что отвечает класс TransactionProcessor,
какие на него возложены обязанности?

Подсказка:
- detect fraud
- send alert

Создайте класс FraudDetector с методом:

    boolean isFraud(Transaction t)

и перенесите всю логику обнаружения мошенничества из класса TransactionProcessor
в класс FraudDetector. Теперь ваш класс FraudDetector служит только для одной цели - обнаружение мошенничества.
Класс FraudDetector следует правилу Single Responsibility Principle, которое
является ключевым в разработке объектно ориентированных приложений.

Создайте в классе TransactionProcessor конструктор и передайте ссылку на FraudDetector
через конструктор.

Исправьте тесты!
