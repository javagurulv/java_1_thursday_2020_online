Система обнаружения мошенничества из level_4.
Экстремальное применение Single Responsibility Principle.
Создайте функциональный интерфейс:

interface FraudRule {

    boolean isFraud(Transaction t);

}

Для каждого правила обнаружения мошеничества создайте класс, который имплементит
интерфейс FraudRule и перенесите в этот новый класс код для этого правила.

Измените класс FraudDetector так, что бы он принимал в конструктор список или массив FraudRule,
а в своём методе isFraud(...) проходил циклом по всем правилам и проверял транзакцию на
предмет мошеничества.

Не поленитесь написать юнит тесты на каждое правило и исправить тесты для класса FraudDetector!
